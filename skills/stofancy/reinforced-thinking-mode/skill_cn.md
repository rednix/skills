# 强化思考模式

## 何时使用

当用户需要深度、多角度分析时激活：
- 关键词："深度思考"、"多角度"、"全面评估"、"设计方案"、"架构规划"
- 意图：复杂系统设计、战略决策、创新、风险评估

**复杂度 → 轮数：**
- 简单（事实性、明确）：2-3 轮
- 中等（功能设计、权衡）：4-5 轮
- 复杂（架构、战略）：5-7 轮
- 棘手（未定义、冲突）：7-10 轮

## 核心原则

**N 次独立的、完整的思考会话——而非迭代完善。**

每轮必须达到：*"如果这是最后一轮，我完全满意。"*

### 心智模式

✅ **思考：** "这是我唯一的机会——全力以赴"

❌ **绝不想：** "下一轮再改进这个"

---

## 为什么文件读写很重要

如果不写文件，AI 会在内部"幻觉"完成 N 轮思考——只是生成摘要而非真正思考。

**文件读写强制执行：**
- 每轮从头思考（无法依赖记忆）
- 真正的独立性（必须读取文件）
- 可验证性（用户看到每轮过程）

---

## 硬性约束

### 文件访问
- **每轮只读取：** `problem.md` + `round_{i-1}.md`（第1轮：仅 problem.md）
- **绝不**读取其他轮次的详细内容

### 禁用词汇
方案文件中绝不使用：`TODO`、`待完善`、`下一轮`、`稍后`、`再细化`

### 信息缺口
- 事实不确定 → 立即搜索
- 需求不确定 → 立即询问用户
- **绝不**假设并继续

---

## 执行流程（详细步骤）

### 阶段 1：初始化（必须执行）

1. **评估问题复杂度**
   - 读取用户问题
   - 确定轮数：简单2-3轮，中等4-5轮，复杂5-7轮，棘手7-10轮

2. **创建工作目录**
   ```bash
   mkdir -p reinforced-thinking
   ```

3. **写入问题定义文件**
   创建 `reinforced-thinking/problem.md`，包含：
   - 问题背景
   - 当前系统/环境状态
   - 核心问题
   - 约束条件
   - 成功标准
   - 复杂度评估理由

---

### 阶段 2：迭代（每轮必须执行）

对于第 X 轮，按顺序执行以下步骤：

#### 步骤 2.1：读取文件
```bash
# 读取问题定义
read reinforced-thinking/problem.md

# 读取上一轮方案（第1轮跳过）
read reinforced-thinking/round_{X-1}.md
```

#### 步骤 2.2：重置思维
在开始写之前，明确告诉自己：
- "我要从**全新角度**思考这个问题"
- "我不能复制上一轮的方法"
- "如果这是**最后一轮**，我必须给出完整方案"

#### 步骤 2.3：自由选择角度
- 批判式思维审视上一轮产方案
- 必要时选择一个与上一轮**明显不同**的角度
- 必要时完全推翻上一轮.

#### 步骤 2.4：全力以赴思考并写入
创建 `reinforced-thinking/round_X.md`，**必须包含以下结构**：

```markdown
# 第 X 轮

## 独立思考宣言
这是第 X 轮，如果这是最后一轮，我对这个方案完全满意。

---

## 核心洞察
（你发现了什么本质问题？1-2句话）

## 方案设计
（详细描述完整方案，包含具体步骤、代码示例、配置等）

## 预期结果
（方案实施后的效果）

## 风险与对策
（潜在风险 + 应对措施）

## 为什么这个方案完整？
（解释为什么这个方案可以直接实施，不需要"后续完善"）
```

#### 步骤 2.5：自我审视（质量检查）
**在保存文件前，必须检查**：

- [ ] **最终质量测试**：如果这是最后一轮，我对这个方案满意吗？
- [ ] **零TODO检查**：文中是否有 `TODO`、`待完善`、`下一轮`？
- [ ] **信息完整检查**：所有需要的细节都写了吗？
- [ ] **可执行性检查**：方案可以直接实施吗？
- [ ] **独立思考检查**：是否基于 problem.md + 仅上一轮？

**如果任何检查不通过 → 重做步骤 2.4**

#### 步骤 2.6：保存文件
```bash
write reinforced-thinking/round_X.md
```

---

### 阶段 3：合成（必须执行）

1. **读取所有轮次**
   ```bash
   read reinforced-thinking/problem.md
   read reinforced-thinking/round_1.md
   read reinforced-thinking/round_2.md
   # ... 读取所有轮次
   ```

2. **分析所有方案**
   - 识别每个方案的独特价值
   - 识别方案之间的冲突和互补
   - 选择最佳方案或组合方案

3. **生成最终报告**
   创建 `reinforced-thinking/final_report.md`：

   ```markdown
   # 最终报告

   ## 问题重述
   （一句话概括问题）

   ## 各轮方案摘要
   - R1: 角度 - 核心方案 - 优势
   - R2: 角度 - 核心方案 - 优势
   - ...

   ## 方案对比
   | 方案 | 优势 | 劣势 | 适用场景 |

   ## 推荐方案
   （选择最佳方案或组合，并解释理由）

   ## 实施计划
   （具体实施步骤）

   ## 风险与注意事项
   ```

---

### 阶段 4：清理（必须执行）

**生成最终报告后，必须询问用户是否清理中间文件。**

1. **列出生成的文件**
   ```bash
   ls -la reinforced-thinking/
   ```

2. **明确询问用户**
   
   向用户展示：
   ```
   强化思考完成！生成了以下文件：
   
   📄 final_report.md    (最终报告 - 保留)
   📄 problem.md          (问题定义)
   📄 round_1.md          (第1轮方案)
   📄 round_2.md          (第2轮方案)
   ...
   
   是否清理中间文件（round_*.md, review_*.md, problem.md）？
   只保留 final_report.md
   ```

3. **等待用户确认**
   
   - 如果用户回答 **是/YES/清理**：删除中间文件
     ```bash
     rm reinforced-thinking/problem.md
     rm reinforced-thinking/round_*.md
     rm reinforced-thinking/review_*.md
     ```
   - 如果用户回答 **否/NO/保留**：保留所有文件

4. **绝不自动清理** — 必须等待用户决定

## 质量保证机制

### 强制重做规则

如果以下任一条件触发，**必须重做当前轮**：

1. **包含禁用词汇**：`TODO`、`待完善`、`下一轮`、`稍后`、`再细化`
2. **不是最终质量**：自己不满意，需要"后续补充"
3. **依赖非授权文件**：读取了 problem.md 和 round_{X-1}.md 以外的文件
4. **方案不完整**：缺少核心部分（问题分析、方案、预期结果、风险）

### 检查清单（每轮必做）

写完每轮后，逐项核对：

```
□ 我选择了与上一轮不同的角度
□ 我没有复制上一轮的方案
□ 方案中没有 TODO、待完善等词汇
□ 如果这是最后一轮，我会满意
□ 所有需要的细节都已包含
□ 方案可以直接实施，不需要"后续开发"
□ 只读取了 problem.md 和上一轮
```

## 常见错误与纠正

| 错误 | 示例 | 纠正方法 |
|------|------|----------|
| 携带上下文 | "基于R1的UX和R2的技术..." | 只写"基于 problem.md 和 R3 的方法..." |
| 留下TODO | "细节下一轮再设计" | 现在就给出核心设计细节 |
| 假设事实 | "用户可能想要X" | 搜索确认或询问用户 |
| 预设方向 | "下一轮：安全角度" | 不预设，让每轮自由选择 |
| 迭代思维 | "在R1基础上改进" | 每轮独立，从头思考 |
| 不写细节 | "详见相关文档" | 直接在文件中写出完整细节 |

---

## 最佳实践

- **发散思维**：无角度矩阵——自由选择任何视角
- **真正独立**：只读 problem.md + 上一轮，而非全部历史
- **每轮全力以赴**：不为"下次迭代"保留实力
- **透明公开**：在文件中说明选择的视角和理由
- **收敛判断**：如连续2轮缺乏真正创新，可建议提前结束
